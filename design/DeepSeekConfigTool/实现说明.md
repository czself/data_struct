# DeepSeek 实验配置工具实现说明

## 1. 项目概述

本项目是一个基于 Qt Widgets 开发的桌面小工具，主要用于调用 DeepSeek 聊天接口。程序提供图形界面，用户可以填写 API 地址和 API Key，选择包含 `.docx` 实验报告的文件夹，输入提示词，然后把提示词发送给 DeepSeek API，最终把返回结果显示在日志窗口中。

项目采用 C++ 和 Qt 实现，使用 qmake 进行构建。整体结构比较简单，主要分为界面层和网络请求层。

## 2. 项目文件结构

```text
DeepSeekConfigTool/
├── DeepSeekConfigTool.pro   # qmake 项目配置文件
├── main.cpp                 # 程序入口，设置应用主题并显示主窗口
├── mainwindow.h             # 主窗口类声明
├── mainwindow.cpp           # 主窗口界面和交互逻辑
├── deepseekapi.h            # DeepSeekAPI 类声明
├── deepseekapi.cpp          # DeepSeek API 请求和响应处理
└── README.md                # 项目简介和运行方式
```

## 3. 核心模块说明

### 3.1 main.cpp

`main.cpp` 是程序入口，主要完成三件事：

1. 创建 `QApplication` 对象。
2. 调用 `applyDarkTheme()` 设置深色主题。
3. 创建并显示 `MainWindow` 主窗口。

其中 `applyDarkTheme()` 使用 Qt 的 `Fusion` 风格，并通过 `QPalette` 设置窗口、文字、按钮、高亮等颜色，让整个程序界面保持统一的深色风格。

程序启动流程如下：

```text
main()
  -> 创建 QApplication
  -> 设置应用名称
  -> 应用深色主题
  -> 创建 MainWindow
  -> 显示窗口
  -> 进入 Qt 事件循环
```

### 3.2 MainWindow 界面层

`MainWindow` 负责界面展示和用户交互，主要代码位于 `mainwindow.h` 和 `mainwindow.cpp`。

主窗口中包含以下控件：

```text
API 地址输入框
API Key 输入框
文件夹选择框
.docx 文件列表
提示词输入框
发送按钮
重置按钮
日志显示框
```

界面通过 `setupUI()` 动态创建，没有使用 `.ui` 文件。这样做的优点是所有界面逻辑都在 C++ 代码中，结构直观，便于学习 Qt 控件和布局。

主要成员变量如下：

```cpp
QLineEdit   *m_apiUrlEdit;
QLineEdit   *m_apiKeyEdit;
QLineEdit   *m_folderEdit;
QPushButton *m_browseBtn;
QListWidget *m_fileList;
QTextEdit   *m_promptEdit;
QPushButton *m_sendBtn;
QPushButton *m_resetBtn;
QTextEdit   *m_logEdit;
DeepSeekAPI *m_api;
```

其中 `m_api` 是后端网络请求对象，界面层通过它把用户输入的提示词发送给 DeepSeek。

### 3.3 DeepSeekAPI 网络层

`DeepSeekAPI` 负责和 DeepSeek API 通信，主要代码位于 `deepseekapi.h` 和 `deepseekapi.cpp`。

它内部使用 Qt 的 `QNetworkAccessManager` 发送 HTTP POST 请求，使用 `QJsonObject`、`QJsonArray`、`QJsonDocument` 构造 JSON 请求体，并解析 DeepSeek 返回的 JSON 数据。

核心成员变量如下：

```cpp
QNetworkAccessManager *m_manager;
QString m_apiUrl;
QString m_apiKey;
```

核心方法如下：

```cpp
void setApiUrl(const QString &url);
void setApiKey(const QString &key);
void sendPrompt(const QString &prompt);
void generateComment(const QString &systemPrompt, double score, const QString &studentText);
QNetworkRequest createRequest() const;
void onReplyFinished(QNetworkReply *reply);
```

其中当前界面主要调用的是 `sendPrompt()`。`generateComment()` 预留了根据系统提示词、成绩和实验报告内容生成评语的功能。

## 4. 功能实现流程

### 4.1 选择文件夹

用户点击“选择文件夹”按钮后，会触发：

```cpp
MainWindow::onBrowseFolder()
```

该函数通过 `QFileDialog::getExistingDirectory()` 打开文件夹选择窗口。用户选择目录后，程序使用 `QDirIterator` 递归遍历该目录下的 `.docx` 文件，并把文件名显示到 `QListWidget` 中。

实现流程如下：

```text
点击选择文件夹
  -> 打开文件夹选择对话框
  -> 获取目录路径
  -> 清空旧文件列表
  -> 递归查找 .docx 文件
  -> 排序
  -> 显示到文件列表
  -> 在日志中显示找到的数量
```

### 4.2 发送提示词

用户点击“发送”按钮后，会触发：

```cpp
MainWindow::onSend()
```

该函数首先读取并检查三个输入：

```text
API 地址
API Key
提示词
```

如果 API 地址、API Key 或提示词为空，程序会在日志窗口中提示用户补充信息。如果输入合法，就调用：

```cpp
m_api->setApiUrl(apiUrl);
m_api->setApiKey(apiKey);
m_api->sendPrompt(prompt);
```

同时程序会暂时禁用发送按钮，避免用户在请求未完成时重复发送。

### 4.3 构造 API 请求

`DeepSeekAPI::sendPrompt()` 会构造符合 DeepSeek 聊天接口格式的 JSON 请求体：

```json
{
  "model": "deepseek-chat",
  "messages": [
    {
      "role": "user",
      "content": "用户输入的提示词"
    }
  ],
  "stream": false,
  "max_tokens": 1024
}
```

请求地址由 API 地址和接口路径拼接而成：

```text
https://api.deepseek.com/chat/completions
```

请求头中包含两项关键内容：

```text
Content-Type: application/json
Authorization: Bearer 用户的 API Key
```

最后通过 `QNetworkAccessManager::post()` 发送 POST 请求。

### 4.4 处理 API 响应

网络请求完成后，Qt 会触发 `QNetworkAccessManager::finished` 信号，并调用：

```cpp
DeepSeekAPI::onReplyFinished(QNetworkReply *reply)
```

该函数主要处理三类情况：

1. 网络错误：例如连接失败、地址错误、Key 无效导致请求异常。
2. JSON 错误：返回内容不是合法 JSON。
3. API 错误：返回内容中包含 `error` 字段。

如果响应正常，程序会读取：

```text
choices[0].message.content
```

这就是 DeepSeek 生成的文本内容。随后通过信号发送给主窗口：

```cpp
emit commentReady(content.trimmed());
```

主窗口收到 `commentReady` 信号后，调用 `onApiResponse()`，把回复内容追加到日志窗口，并重新启用发送按钮。

如果发生错误，则通过：

```cpp
emit errorOccurred(errorText);
```

主窗口收到错误后调用 `onApiError()`，把错误信息显示到日志窗口。

## 5. 信号与槽机制

本项目使用 Qt 的信号与槽机制连接界面和网络层。

主窗口中连接按钮事件：

```cpp
connect(m_browseBtn, &QPushButton::clicked, this, &MainWindow::onBrowseFolder);
connect(m_sendBtn,   &QPushButton::clicked, this, &MainWindow::onSend);
connect(m_resetBtn,  &QPushButton::clicked, this, &MainWindow::onReset);
```

主窗口中连接 API 响应事件：

```cpp
connect(m_api, &DeepSeekAPI::commentReady,
        this, &MainWindow::onApiResponse);
connect(m_api, &DeepSeekAPI::errorOccurred,
        this, &MainWindow::onApiError);
```

网络层中连接请求完成事件：

```cpp
connect(m_manager, &QNetworkAccessManager::finished,
        this, &DeepSeekAPI::onReplyFinished);
```

这样设计后，界面层只负责用户操作和结果显示，网络层只负责请求和解析，两部分职责比较清晰。

## 6. 重置和日志功能

点击“重置”按钮后会调用：

```cpp
MainWindow::onReset()
```

该函数会清空 API Key、文件夹路径、提示词、文件列表和日志，同时把 API 地址恢复为默认值：

```text
https://api.deepseek.com
```

日志功能由 `logMessage()` 实现。每条日志都会加上当前时间：

```text
[12:30:05] 正在发送请求...
```

日志不仅会显示在界面中的文本框里，也会通过 `qDebug()` 输出到调试控制台，方便开发时查看。

## 7. 构建配置

`DeepSeekConfigTool.pro` 是 qmake 项目配置文件，内容中声明了项目需要的 Qt 模块：

```pro
QT += core gui widgets network
```

其中：

```text
core    提供 Qt 基础类
gui     提供图形界面基础能力
widgets 提供 QWidget、QPushButton、QTextEdit 等控件
network 提供 QNetworkAccessManager 等网络请求类
```

项目使用 C++17：

```pro
CONFIG += c++17
```

源码和头文件通过 `SOURCES`、`HEADERS` 加入构建。

## 9. 总结

本项目的实现思路可以概括为：

```text
Qt Widgets 构建界面
  -> 用户填写 API 信息和提示词
  -> MainWindow 检查输入并调用 DeepSeekAPI
  -> DeepSeekAPI 构造 JSON 请求
  -> QNetworkAccessManager 发送 HTTP POST
  -> 解析返回 JSON
  -> 通过信号把结果传回界面
  -> 日志窗口显示回复或错误
```

它的重点在于把界面交互、网络请求和结果处理分开实现，既方便理解，也方便后续扩展。例如以后可以继续增加读取 `.docx` 内容、批量生成实验报告评语、导出结果等功能。

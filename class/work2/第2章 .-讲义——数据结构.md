## 2.指针返回

指针返回的关键是，调用函数依然能找到这个指针。

像下面的代码就是万万不能用的，大家要思考清楚为什么。这样学习才会进步。

```c
char * generate(){
	char *p = "123";
	return p;
}
```

有两种修改方式。

- 一种是动态内存分配。这种内存分配之后，一定要告诉调用者，要free。

```c
char * generate(){
	char *p = (char*) malloc(sizeof(char) * 3);  
    memset(p,97,sizeof(char) * 3); //得到字符串 “aaa”
	return p;
}

int main(){
    char * pointer = generate();
    free(pointer);
}

/**
* 要补充几点：
* 字符串“aaa”是存储在堆上的。
* p指针所占据的内存空间在generate函数的局部区域，随着函数的结束，可能会被覆盖。
* pointer所占据的内存空间在main函数的局部区域
* 
* 这一点在课堂上没讲，我怕把问题搞复杂了。有兴趣的同学可自行深入学习。
*/
```

- 第二种修改方式。调用者传入指针。这个指针既是参数又成了返回值，为什么指针能修改，也要想清楚。

```c
void generate(char* c ,int n){
	//对c进行修改
    c[0] = '0';
	return;
}

int main(){
    char * c = "123";
    generate(c,3); // 变成”023“
    return 0;
}
```
